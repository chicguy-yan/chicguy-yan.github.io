Research OpenClaw

面向材料 / 化学 / 环境 / 生物等实验场景的透明 AI Agent 工作台， 目标是把长周期科研任务流做成 可追溯、可验证、可回放 的产品系统。

项目摘要

这个项目不是简单套壳聊天，而是对真实科研场景做了结构性重构： 用文件系统替代黑盒向量库，把“AI 记住了什么、为什么这么建议、证据链从哪里来”变成产品级可见能力。

产品问题 180 天实验周期里，上下文、证据与阶段成果容易断链
核心解法 三层 Markdown 记忆系统 + Context Orchestrator + Trace 回放
技术实现 FastAPI、LangChain、LangGraph、React、TypeScript、SSE
产品价值 把“科研助手”做成可审计工作台，而不是不可解释的黑盒问答
把科研场景拆解成 Identity / Timeline / Atom Notes 三层记忆结构，适配真实研究主线推进。
设计显式上下文注入逻辑，解决长周期任务里“该带哪些信息进模型”的产品与工程问题。
前端采用三栏 IDE 风格布局，把文件树、聊天流与 Trace 审计面板并置，强化可信感。
支持多工作区、多会话、技能插件与 5 个核心工具，让系统具备进一步扩展成 AI 工作平台的骨架。

TAP · 数学核心图式构建器

一个面向高考数学场景的 AI 教育产品原型， 试图把“错题整理”升级为 Trigger / Action / Pitfall 的思维模型训练系统。

核心机制

传统错题本只记录题和答案，难以沉淀迁移能力。TAP 的产品核心是把一次错误提炼成可复用的思维图式， 让学生不是单次纠错，而是逐步建立“看到条件就触发模型”的解题反应。

目标用户 需要提高标准化题型命中率与复盘效率的高中生
产品结构 拍题入口、分步脚手架、图式卡片、模型库、错题本
实现方式 React 19、TypeScript、Vite、组件化交互原型
核心亮点 把抽象“学习策略”翻译成可视化、可复盘、可持续积累的产品语言
围绕“80% 分数来自标准化高频模型”设计产品叙事，先建立用户认知，再进入交互流程。
把一次题目分析拆成 Trigger、Action、Pitfall 三部分，强化可迁移的心智模型。
通过图式模型库、错题状态与最近坑点列表，构建连续学习而非单次问答的体验。
用前端原型形式验证教育 AI 的交互结构，为后续接入识题、多模态分析和个性化推荐留出空间。
